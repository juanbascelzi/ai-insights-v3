================================================================================
  Plan: Sistema de Analisis de Transcripts - Humand Sales Insights v2
================================================================================

CONTEXT
-------
Humand necesita extraer insights normalizados de 500+ transcripts de llamadas
de ventas almacenados en una view de Supabase. Los insights deben ser
categorizados segun una taxonomia HR completa para permitir analisis SQL y
dashboards BI. Se usa la API de OpenAI (ChatGPT). Es un MVP one-shot.

Prerequisito pendiente: Credenciales de Supabase (URL + service key) y
estructura de la view.

================================================================================
  2. SCHEMA DE BASE DE DATOS
================================================================================

2.1 TABLAS DE REFERENCIA (TAXONOMIA)
--------------------------------------
La taxonomia vive en tablas de referencia en Supabase. El script las puebla
en el setup. Sirven para validacion, JOINs en dashboards, y extensibilidad.

  tax_hr_categories     (code PK, display_name, sort_order)
  tax_modules           (code PK, display_name, hr_category FK, status, sort_order)
  tax_pain_subtypes     (code PK, display_name, description, theme, module FK NULL, sort_order)
  tax_deal_friction_subtypes  (code PK, display_name, description)
  tax_faq_subtypes      (code PK, display_name, description)
  tax_competitive_relationships (code PK, display_name, description)
  tax_competitors       (name PK, region)
  tax_feature_names     (code PK, display_name, suggested_module FK, is_seed, created_at)


2.2 TABLA PRINCIPAL
--------------------
  transcript_insights:
    - Solo codigos, sin display names duplicados
    - Display names se resuelven via JOINs o la VIEW v_insights_dashboard
    - module es obligatorio para product_gap
    - content_hash para dedup (UNIQUE index)


2.3 VISTA PARA DASHBOARDS
---------------------------
  v_insights_dashboard:
    - JOINs a todas las tablas de referencia
    - Resuelve: insight_type_display, insight_subtype_display,
      module_display, module_status, hr_category, hr_category_display,
      pain_theme, pain_scope, feature_display, feature_is_seed,
      competitor_relationship_display

  Ver PLAN.md para el SQL completo de todas las tablas y la vista.


================================================================================
  3. ARQUITECTURA DEL SCRIPT
================================================================================

ESTRUCTURA DE ARCHIVOS
  main.py              Entry point con CLI (--dry-run, --sample N, --resume)
  config.py            Variables de entorno, constantes
  taxonomy.py          Modulos, categorias, pains, features, competidores
  models.py            Pydantic models para Structured Output de OpenAI
  chunker.py           Chunking token-aware con tiktoken
  prompt_builder.py    System prompt con taxonomia completa + few-shot examples
  batch_processor.py   OpenAI Batch API: crear JSONL, submit, poll, download
  parser.py            Validacion, normalizacion, extension de features
  db.py                Cliente Supabase: leer view, escribir insights, gestionar taxonomia
  pipeline.py          Orquestador: fetch -> chunk -> batch -> parse -> load
  schema.sql           DDL completo: tablas de referencia + insights + vista
  seed_taxonomy.py     Script para poblar tablas de referencia con datos iniciales
  state.json           Estado de procesamiento para resume
  .env                 SUPABASE_URL, SUPABASE_KEY, OPENAI_API_KEY
  requirements.txt     openai, supabase, tiktoken, pydantic, tenacity, python-dotenv

PIPELINE
  0. SETUP    -> Crear tablas (schema.sql) + poblar taxonomia (seed_taxonomy.py)
  1. EXTRACT  -> Leer transcripts de Supabase view
  2. CHUNK    -> Tokenizar con tiktoken, chunkar si >12K tokens
  3. BUILD    -> Generar JSONL con prompts + taxonomia
  4. SUBMIT   -> Enviar al OpenAI Batch API
  5. POLL     -> Polling cada 60s hasta completar (~15-60 min)
  6. PARSE    -> Descargar, validar con Pydantic, extender features nuevos
  7. LOAD     -> Upsert a Supabase con dedup por content_hash

COSTOS
  gpt-4o-mini + Batch API  ~$1-2 USD (500 transcripts)  Muy buena calidad
  gpt-4o + Batch API       ~$20-25 USD                  Excelente calidad

EJECUCION
  1. Setup: Crear tablas, poblar taxonomia, configurar .env
  2. Dry run (5 transcripts): gpt-4o, revisar, iterar prompt
  3. Validation (25 transcripts): gpt-4o-mini, comparar calidad
  4. Full batch (500+): Batch API con gpt-4o-mini
  5. Quality audit: Samplear 10%, verificar
  6. Analisis: Queries SQL + dashboards BI

CONFIGURACION AI (OpenAI)
  Modelo: gpt-4o-mini via Batch API (50% descuento)
  Structured Output: JSON Schema de OpenAI (response_format)
  Temperature: 0 (maxima consistencia)
  System prompt: taxonomia completa embebida + 2-3 few-shot examples
  Chunking: 12K tokens max por request, split en speaker turns


================================================================================
  4. FLUJO AUTOMATIZADO END-TO-END
================================================================================

PREREQUISITOS
  - Python 3.10+
  - Cuenta OpenAI con API key (platform.openai.com/api-keys)
  - Proyecto Supabase con view de transcripts
  - Supabase URL + Service Role Key (Settings > API)

SETUP INICIAL (una sola vez)
  cd ai-insights-v3
  python3 -m venv .venv
  source .venv/bin/activate
  pip install -r requirements.txt
  cp .env.example .env       # editar con credenciales
  python main.py setup       # crea tablas + puebla taxonomia

  El comando "setup" automaticamente:
    1. Ejecuta schema.sql (8 tablas referencia + tabla principal + vista)
    2. Ejecuta seed_taxonomy.py (7 categorias, 39 modulos, 87 pains, etc.)
    3. Valida que todo se creo correctamente

EJECUCION DEL PIPELINE
  python main.py run --sample 5 --model gpt-4o         # Dry run
  python main.py run --sample 25 --model gpt-4o-mini    # Validacion
  python main.py run --model gpt-4o-mini                # Full batch
  python main.py run --resume                           # Resumir
  python main.py status                                 # Estado del batch

FLAGS
  --sample N      Solo N transcripts (testing, API directa)
  --model X       gpt-4o o gpt-4o-mini (default: gpt-4o-mini)
  --resume        Retoma desde ultimo checkpoint
  --dry-run       Genera JSONL sin enviar a OpenAI
  (sin flags)     Todos los transcripts via Batch API

QUE PASA INTERNAMENTE
  1. Conecta a Supabase, lee transcripts de la view
  2. Filtra ya procesados (por content_hash)
  3. Chunkea transcripts largos (>12K tokens)
  4. Genera batch_input.jsonl
  5. Sube JSONL a OpenAI, crea batch
  6. Polling cada 60s hasta completar
  7. Descarga resultados, valida con Pydantic
  8. Auto-inserta features nuevas en tax_feature_names
  9. Inserta insights con dedup por content_hash
  10. Actualiza state.json

DONDE CORRERLO

  GitHub Codespace     $0    Si no queres instalar local (60h gratis/mes)

COSTOS TOTALES
